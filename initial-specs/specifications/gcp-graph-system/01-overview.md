# 01. Overview

## Purpose

本仕様書は、ファイルから Gemini を用いて意味構造を抽出し、階層レベルとカテゴリを持つノード、およびその関係をノードエッジ形式で可視化する Web システムの要件、構成、データ設計、API、運用方針を定義する。

初期段階では GCP のサーバレス構成で最小限の実装を行い、将来的に大容量処理、非同期化、検索強化、グラフ探索高度化へ拡張できることを前提とする。

## Goals and Non-Goals

### Goals

- ユーザーがファイルをアップロードできること
- ファイルからテキストを抽出し、Gemini でノード・エッジを生成できること
- ノードを `level` と `category` で構造化して保存できること
- ドキュメント単位でノードグラフを取得し、フロントで可視化できること
- 出典となる chunk を追跡できること
- 初期実装は低運用負荷で立ち上げられること

### Non-Goals

- 初期段階でのリアルタイム共同編集
- 初期段階での大規模ストリーミング処理
- 初期段階での複雑なグラフアルゴリズム実行
- 初期段階での完全自動オントロジー統合

## Users and Use Cases

### Primary Users

- 分析担当者
- 企画担当者
- 文書の構造化・要約を行いたい業務ユーザー

### Primary Use Cases

1. ユーザーがファイルをアップロードする
2. システムがファイルを解析して階層付きノードと関係を生成する
3. ユーザーがグラフを閲覧する
4. ユーザーがノードをクリックし、説明と出典を確認する
5. 将来的にユーザーが複数ドキュメント横断で概念を探索する

## Release Scope Summary

### Initial Release

- 単一ファイルアップロード
- PDF、Markdown、TXT、CSV 対応
- Gemini による階層付きノード抽出
- BigQuery 保存
- document 単位グラフ表示
- ノード詳細と出典表示
- フロントは `TypeScript + React Flow`
- バックエンドは `Go + Connect RPC`

### Deferred Items

- 大量ファイル同時処理
- 横断検索
- 高度なアクセス制御
- 高度なグラフ分析
